## 🔥 DESIGN RULES — NON-NEGOTIABLE

### What you MUST do
- **Glassmorphism cards** on dark backgrounds: `BackdropFilter` with `ImageFilter.blur`, semi-transparent border
- **Glow effects** on primary action buttons using `BoxShadow` with `electricBlue` at 40% opacity
- **Staggered animations** on every list — each item slides up with 80ms delay between items
- **Hero transitions** between list items and detail pages — always
- **Custom bottom nav** — floating pill-shaped navbar with glass effect, not the default Flutter BottomNavigationBar
- **Shimmer skeletons** that match the exact shape of the content they replace
- **Micro-animations** on every button (scale 0.96 on press using `GestureDetector` + `AnimationController`)
- **Number counters** — all statistics animate from 0 to final value on screen entry
- **Gradient text** for hero headings using `ShaderMask`
- **Pull-to-refresh** with a custom Lottie animation (not the default spinner)
- **Empty states** with custom Lottie animations + contextual copy
- **Role-specific color tinting** — Admin screens tinted gold, Teacher tinted teal, Student tinted blue

### What you must NEVER do
- ❌ Default AppBar (blue with white text) — EVER
- ❌ Default `CircularProgressIndicator` — replace with custom Lottie
- ❌ `ListTile` with default styling — always custom
- ❌ Flat white cards with no depth
- ❌ System font (Roboto) for ANY visible text
- ❌ Square corners anywhere
- ❌ Default `SnackBar` — use custom toast from bottom
- ❌ Default `AlertDialog` — use custom bottom sheets
- ❌ Default `Switch` or `Checkbox` — always custom styled
- ❌ Same color scheme for all 4 roles — each role has its own accent

## 🔄 STATE MANAGEMENT PATTERNS (BLoC)

```dart
// EVERY feature follows this exact pattern:

// 1. EVENTS (what can happen)
abstract class FeatureEvent extends Equatable {}
class FeatureLoaded extends FeatureEvent {}
class FeatureRefreshed extends FeatureEvent {}
class FeatureItemSelected extends FeatureEvent {
  final String id;
  const FeatureItemSelected(this.id);
  @override List<Object> get props => [id];
}

// 2. STATES (what the UI can show)
abstract class FeatureState extends Equatable {}
class FeatureInitial extends FeatureState {}
class FeatureLoading extends FeatureState {}
class FeatureLoadingMore extends FeatureState { // For pagination
  final List<dynamic> currentItems;
}
class FeatureLoaded extends FeatureState {
  final List<dynamic> items;
  final bool hasMore;
  final int totalCount;
}
class FeatureEmpty extends FeatureState {}
class FeatureError extends FeatureState {
  final String message;
  final bool isNetworkError;
}

// 3. BLOC
class FeatureBloc extends Bloc<FeatureEvent, FeatureState> {
  FeatureBloc({required this.usecase}) : super(FeatureInitial()) {
    on<FeatureLoaded>(_onLoaded);
    on<FeatureRefreshed>(_onRefreshed);
  }

  Future<void> _onLoaded(FeatureLoaded event, Emitter emit) async {
    emit(FeatureLoading());
    final result = await usecase(NoParams());
    result.fold(
      (failure) => emit(FeatureError(
        message: failure.message,
        isNetworkError: failure is NetworkFailure,
      )),
      (items) => items.isEmpty
          ? emit(FeatureEmpty())
          : emit(FeatureLoaded(items: items, hasMore: false, totalCount: items.length)),
    );
  }
}

// 4. UI PATTERN
BlocBuilder<FeatureBloc, FeatureState>(
  builder: (context, state) {
    return switch (state) {
      FeatureInitial()   => const SizedBox.shrink(),
      FeatureLoading()   => const FeatureShimmer(),    // Custom shimmer
      FeatureEmpty()     => const CPEmptyState(...),
      FeatureError()     => CPErrorState(
                              message: state.message,
                              onRetry: () => context.read<FeatureBloc>()
                                  .add(FeatureLoaded()),
                            ),
      FeatureLoaded()    => FeatureList(items: state.items),
      _                  => const SizedBox.shrink(),
    };
  },
)
```
